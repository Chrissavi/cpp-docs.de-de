# <a name="c26409-no_new_delete"></a>C26409 NO_NEW_DELETE

Auch wenn der Code für Aufrufe von malloc () und Free () sauber ist, wird empfohlen, dass Sie bessere Optionen als die explizite Verwendung von Operatoren " [New" und "Delete](/cpp/cpp/new-and-delete-operators)" in Erwägung nehmen.

  **C++ Core Guidelines**: [R. 11: vermeiden Sie das explizite Aufrufen von New und DELETE](https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines#r11-avoid-calling-new-and-delete-explicitly) .

Die ultimative Lösung ist die Verwendung von intelligenten Zeigern mit geeigneten Factory-Funktionen, wie z. b. " [Std:: make_unique](/cpp/standard-library/memory-functions#make_unique)".

## <a name="remarks"></a>Bemerkungen

- Die Prüfung warnt bei Aufrufen jeder Art von Operator **`new`** oder **`delete`** : Scalar, Vector, überladene Versionen (Global und klassenspezifisch) sowie bei Platzierungs Versionen. Der letztere Fall erfordert möglicherweise einige Erläuterungen zu den wichtigsten Richtlinien in Bezug auf Empfohlene Korrekturen und kann in Zukunft ausgelassen werden.
