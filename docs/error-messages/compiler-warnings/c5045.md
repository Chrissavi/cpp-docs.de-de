# <a name="compiler-warning-c5045"></a>Compilerfehler Warnung C5045

> Compilerfehler wird Absorptionsspektrum Entschärfung einfügen, Arbeitsspeicherlast angegeben, wenn /Qspectre wechseln

## <a name="remarks"></a>Hinweise

Warnung C5045 können Sie sehen, welche Bitmuster in Ihrem Code linderungsmaßnahme Absorptionsspektrum z. B. ein LFENCE dazu führen, dass wenn einzufügenden der [/Qspectre](../../build/reference/qspectre.md) -Compileroption angegeben ist. Dadurch können Sie identifizieren, welche Codedateien von das Sicherheitsproblem betroffen sind. Diese Warnung dient nur Informationszwecken: das Minderung wird nicht eingefügt werden, bis Sie neu, verwenden Kompilieren die **/Qspectre** wechseln. Die Funktionalität des C5045 ist unabhängig von der **/Qspectre** zu wechseln, sodass Sie sie sowohl in der gleichen Kompilierung verwenden können.

Diese Warnung ist neu in Visual Studio 2017 Version 15.7 und ist standardmäßig deaktiviert. Verwendung [/Wall](../../build/reference/compiler-option-warning-level.md) alle Warnungen zu aktivieren, die standardmäßig deaktiviert sind oder __/w__*n*__5038__ C5045 als eine Ebene aktivieren *n* Warnung. In der IDE ist die standardmäßigen Warnstufe **/W3** und diese Warnung kann aktiviert werden, in das Projekt **Eigenschaftenseiten** Dialogfeld. Open **Konfigurationseigenschaften** > **C/C++-** > **Befehlszeile** und klicken Sie in der **Zusatzoptionen** Fügen */w35045*, wählen Sie dann **OK**. Weitere Informationen finden Sie unter [compilerwarnungen, die standardmäßig deaktiviert sind](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Informationen zum Deaktivieren von Warnungen nach Compilerversion finden Sie unter [compilerwarnungen von Compilerversion](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Beispiel

Im folgende Beispiel löst die Warnung C5045 beim Kompilieren von Visual Studio 2017 Version 15.7 mit der **/Wall** oder **/w35045** und **/W3** Optionen:

```cpp
// C5045.cpp
// Compile with: cl /EHsc /W3 /w35045 C5045.cpp

int G, G1, G2;

__forceinline
int * bar(int **p, int i)
{
    return p[i];
}
 
__forceinline
void bar1(int ** p, int i)
{
    if (i < G1) {
        auto x = p[i]; // C5045: mitigation here
        G = *x;
    }
}

__forceinline
void foo(int * p)
{
    G = *p;
}

void baz(int ** p, int i)
{
    if (i < G1) {
        foo(bar(p, i + G2));
    }
    bar1(p, i);
}

int main() { }
```

Die Compilerausgabe, wenn die Warnung aktiviert ist, sieht etwa wie folgt:

```Output
C:\Users\username\source\repos\C5045>cl /W3 /w35045 C5045.cpp
Microsoft (R) C/C++ Optimizing Compiler Version 19.14.26431 for x86
Copyright (C) Microsoft Corporation.  All rights reserved.

C5045.cpp
c:\users\username\source\repos\c5045\c5045.cpp(16) : warning C5045: Compiler will insert Spectre mitigation for memory load if /Qspectre switch specified
c:\users\username\source\repos\c5045\c5045.cpp(15) : note: index 'i' range checked by comparison on this line
c:\users\username\source\repos\c5045\c5045.cpp(17) : note: feeds memory load on this line
Microsoft (R) Incremental Linker Version 14.14.26431.0
Copyright (C) Microsoft Corporation.  All rights reserved.

/out:C5045.exe
C5045.obj
```

Die warnungsmeldungen zeigen, dass eine Entschärfung würde in Zeile 16 eingefügt wurden. Er notiert sich auch, dass das Minderung benötigt wird, da der Index "i" in Zeile 15 die Auslastung des Arbeitsspeichers in Zeile 17 feeds. Die Spekulatives erfolgt über Balken- und bar1, aber das Minderung wirkungsvoll in Zeile 16 platziert wird.
