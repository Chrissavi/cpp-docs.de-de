# <a name="building-on-the-framework"></a>Erstellen im Framework

Die Rolle beim Konfigurieren einer Anwendung mit dem MFC-Framework besteht darin, den anwendungsspezifischen Quellcode bereitzustellen und die Komponenten zu verbinden, indem Sie definieren, auf welche Nachrichten und Befehle Sie Antworten. Verwenden Sie die Methoden C++ Language und Standard C++, um eigene anwendungsspezifische Klassen von den von der Klassenbibliothek bereitgestellten zu leiten und das Verhalten der Basisklasse zu überschreiben und zu erweitern.

In den zugehörigen Themen beschreiben die folgenden Tabellen die allgemeine Abfolge von Vorgängen, die Sie in der Regel befolgen, und ihre Zuständigkeiten und die Zuständigkeiten des Frameworks:

- [Sequenz zum Entwickeln einer Anwendung mit dem Framework](sequence-of-operations-for-building-mfc-applications.md)

- [Reihenfolge der Operationen zur Erstellung von OLE-Anwendungen](sequence-of-operations-for-creating-ole-applications.md)

- [Operationssequenz zur Erstellung von ActiveX-Steuerelementen](sequence-of-operations-for-creating-activex-controls.md)

- [Reihenfolge der Operationen zur Erstellung Datenbankanwendungen](sequence-of-operations-for-creating-database-applications.md)

In den meisten Fällen können Sie diese Tabellen als Sequenz von Schritten zum Erstellen einer MFC-Anwendung befolgen, obwohl einige der Schritte alternative Optionen sind. Die meisten Anwendungen verwenden z. b. eine Art von Ansichts Klasse aus den verschiedenen verfügbaren Typen.
