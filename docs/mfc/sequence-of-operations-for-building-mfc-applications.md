# <a name="sequence-of-operations-for-building-mfc-applications"></a>Reihenfolge der Operationen zur Erstellung von MFC-Anwendungen

In der folgende Tabelle wird erläutert, die allgemeine Abfolge, die Sie in der Regel befolgen können, wie Sie die MFC-Anwendung entwickeln.

### <a name="sequence-for-building-an-application-with-the-framework"></a>Sequenz zum Erstellen einer Anwendung mit dem Framework

|Aufgabe|Sie tun|Das Framework ermöglicht|
|----------|------------|------------------------|
|Erstellen Sie eine Skelette-Anwendung.|Führen Sie die [MFS-Anwendungsassistenten](../mfc/reference/mfc-application-wizard.md). Geben Sie die gewünschten Optionen auf den Optionsseiten. Optionen sind, sodass Sie eine COM-Komponente, Container oder beides: Hinzufügen von Automation; und nehmen die Anwendung Datenbank unterstützen.|Die MFC-Anwendungs-Assistent erstellt die Dateien für eine Skelett-Anwendung, einschließlich der Quelldateien für Ihre Anwendung, Dokument, Ansicht und Rahmenfenster; eine Ressourcendatei; eine Projektdatei; und anderen Benutzern, alle Ihren Anforderungen entsprechend angepasst.|
|Sehen Sie, was das Framework und der MFC-Anwendungs-Assistent bieten, ohne eine Zeile mit Ihren eigenen Code hinzuzufügen.|Erstellen Sie die Skelette-Anwendung, und führen Sie es in Visual C++.|Das ausgeführte Anwendungsgerüst abgeleitet wird, viele **Datei**, **bearbeiten**, **Ansicht**, und **Hilfe** Menübefehle aus dem Framework. Für MDI-Anwendungen erhalten Sie auch einen voll funktionsfähigen Windows-Menü, und das Framework verwaltet die Erstellung, Anordnung und Zerstörung von untergeordneten MDI-Fenster.|
|Erstellen Sie die Benutzeroberfläche Ihrer Anwendung.|Verwenden Sie die Visual C++ [Ressourcen-Editoren](../windows/resource-editors.md) Benutzeroberfläche der Anwendung visuell zu bearbeiten:<br /><br /> – Erstellen von Menüs.<br />-Definieren Sie Accelerators.<br />– Erstellen von Dialogfeldern.<br />– Erstellen und Bearbeiten von Bitmaps, Symbole und Cursor.<br />-Bearbeiten Sie die Symbolleiste, die vom MFC-Anwendungs-Assistenten für Sie erstellt.<br />– Erstellen und Bearbeiten von anderen Ressourcen.<br /><br /> Sie können auch die Dialogfelder der Dialog-Editor testen.|Die vom MFC-Anwendungs-Assistenten erstellte Ressourcendatei stellt viele der Ressourcen, die Sie benötigen. Visual C++ können Sie die vorhandene Ressourcen bearbeiten und neue Ressourcen hinzufügen, einfache und visuelle.|
|Ordnen Sie Handlerfunktionen Menüs.|Verwenden der **Ereignisse** Schaltfläche der [Fenster "Eigenschaften"](/visualstudio/ide/reference/properties-window) für die Verbindung Menüs und Zugriffstasten Handlerfunktionen in Ihrem Code.|Das Fenster "Eigenschaften" fügt Meldungszuordnungseinträge und leere Funktionsvorlagen, in den Quelldateien, die Sie angeben, und viele manuelle Codierungsaufgaben verwaltet.|
|Schreiben Sie Ihren Handlercode für.|Mithilfe der Klassenansicht, um direkt an den Code in der Quellcode-Editor zu wechseln. Geben Sie den Code für die Handlerfunktionen. Weitere Informationen zur Verwendung der Klassenansicht und Assistenten, mit denen einer Projekt Code hinzu, finden Sie unter [Hinzufügen neuer Funktionen mit Code-Assistenten](../ide/adding-functionality-with-code-wizards-cpp.md).|Klassenansicht wird der Editor geöffnet, führt einen Bildlauf zu der empty-Funktion-Vorlage und platziert den Cursor für Sie.|
|Ordnen Sie Symbolleisten-Schaltflächen Befehle.|Ordnen Sie jeder Schaltfläche auf der Symbolleiste zu einem Menü oder die Tastenkombination Befehl durch Zuweisen der Schaltfläche mit dem entsprechenden Befehls-ID.|Das Framework steuert die Zeichnung, aktivieren, deaktivieren, Überprüfung und anderen visuellen Aspekte der Symbolleisten-Schaltflächen.|
|Testen Sie Ihre Handlerfunktionen.|Erstellen Sie die Anwendung neu, und verwenden Sie die integrierten Tools zum Debuggen, um zu testen, ob die Handler ordnungsgemäß ausgeführt.|Sie können Schritt oder verfolgen den Code, um festzustellen, wie die Handler aufgerufen werden. Wenn Sie vom Handlercode ausgefüllt haben, Ausführen die Handler Befehle. Das Framework wird automatisch deaktiviert werden, Menüelemente und Symbolleisten-Schaltflächen, die nicht verarbeitet werden.|
|Hinzufügen [Dialogfelder](../mfc/dialog-boxes.md).|Entwerfen von Dialogfeld-Vorlagenressourcen mit den Dialog-Editor. Dann Erstellen einer Dialogfeldklasse und der Code, der im Dialogfeld behandelt.|Das Framework verwaltet das Dialogfeld und vereinfacht das Abrufen von Informationen, die vom Benutzer eingegeben werden.|
|Initialisieren, zu überprüfen und Abrufen von Daten für das Dialogfeld.|Sie können auch definieren, wie das Dialogfeld-Steuerelemente werden initialisiert und überprüft werden. Verwenden Sie Visual Studio, um die Dialogfeldklasse Membervariablen hinzu, und deren Zuordnung zu Dialogfeld-Steuerelemente. Geben Sie Validierungsregeln auf jedes Steuerelement angewendet werden, wie Daten eingegeben werden. Geben Sie Ihre eigenen benutzerdefinierten Überprüfungen aus, wenn Sie möchten.|Das Framework verwaltet die Dialogfeld-Initialisierung und Validierung. Wenn der Benutzer ungültige Informationen eingibt, wird das Framework zeigt ein Meldungsfeld an, und ermöglicht dem Benutzer die Daten erneut eingeben.|
|Erstellen Sie zusätzliche Klassen.|Mithilfe der Klassenansicht zum Erstellen von zusätzlichen Dokument-, Ansicht und Rahmenfensterklassen hinausgehen vom MFC-Anwendungs-Assistenten automatisch erstellt. Sie können zusätzliche Datenbank-Recordset-Klassen, Klassen usw. erstellen. (In der Klassenansicht, können Sie nicht von MFC-Klassen abgeleitete Klassen erstellen.)|Klassenansicht fügt diese Klassen zu Ihren Quelldateien und hilft Ihnen, ihre Verbindungen Befehle zu definieren, die sie behandeln.|
|Bereit zu verwendenden Komponenten Ihrer Anwendung hinzufügen.|Verwenden der `New Item dialog box` eine Vielzahl von Elementen hinzufügen.|Diese Elemente sind einfach in Ihre Anwendung integrieren und ersparen Ihnen viel Arbeit.|
|Die Dokumentklasse zu implementieren.|Implementieren Sie Ihre anwendungsspezifische Document-Klasse oder Klassen. Hinzufügen von Membervariablen zum Speichern von Datenstrukturen. Fügen Sie die Member-Funktionen, um eine Schnittstelle zu den Daten bereitzustellen.|Das Framework weiß bereits, wie für die Interaktion mit Datendateien des Dokuments. Es kann öffnen und schließen Dokumentdateien, lesen und schreiben die Daten des Dokuments und andere Benutzeroberflächen zu behandeln. Können Sie sich auf wie die Daten eines Dokuments geändert werden.|
|Implementieren Sie öffnen, speichern, und speichern Sie als Befehle.|Schreiben von Code für des Dokuments des `Serialize` Member-Funktion.|Zeigt das Framework Dialogfelder für die **öffnen**, **speichern**, und **speichern unter** von Befehlen auf die **Datei** Menü. Er schreibt und Lesevorgänge Sichern eines Dokuments mithilfe der im angegebenen Datenformat Ihre `Serialize` Member-Funktion.|
|Implementieren Sie Ihre Ansichtsklasse.|Implementieren Sie eine oder mehrere Klassen, Ihre Dokumente entspricht. Implementieren Sie die Ansicht des Member-Funktionen, die Sie mit der Benutzeroberfläche in der Klassenansicht zugeordnet. Eine Vielzahl von [CView](../mfc/reference/cview-class.md)-abgeleitete Klassen sind verfügbar, einschließlich [CListView](../mfc/reference/clistview-class.md) und [CTreeView](../mfc/reference/ctreeview-class.md).|Das Framework verwaltet die meisten der Beziehung zwischen einem Dokument und dessen Ansicht. Memberfunktionen der Ansicht auf der Ansicht-Dokument, um das Image auf dem Bildschirm oder gedruckte Seite zu rendern und beim Aktualisieren des Dokuments Datenstrukturen als Reaktion auf Befehle zum Bearbeiten zugreifen.|
|Verbessern Sie die Standard-drucken.|Wenn Sie das Drucken von mehrseitiges unterstützen möchten, überschreiben Sie Memberfunktionen der Ansicht.|Das Framework unterstützt die **Drucken**, **Seiteneinrichtung**, und **Seitenansicht** von Befehlen auf die **Datei** Menü. Sie müssen es mitteilen, wie Sie das Dokument in mehrere Seiten aufgeteilt.|
|Fügen Sie einen Bildlauf hinzu.|Wenn Sie einen Bildlauf unterstützen müssen, abgeleitet werden, Ihre Ansichtsklasse oder Klassen von [CScrollView](../mfc/reference/cscrollview-class.md).|Die Ansicht fügt automatisch Bildlaufleisten hinzu, wenn das Ansichtsfenster zu klein ist.|
|Erstellen Sie Formularansichten.|Wenn Ihre Ansichten auf Dialogfeld-Vorlagenressourcen basieren soll, leiten Sie Ihre Ansichtsklasse oder Klassen von [CFormView](../mfc/reference/cformview-class.md).|Die Sicht mithilfe der Dialogfeldvorlagen-Ressource Steuerelemente angezeigt. Benutzer kann steuern, in der Ansicht vom Steuerelement Registerkarte.|
|Erstellen Sie Datenbankformulare.|Wenn Sie eine Datenzugriff formularbasierte Anwendung möchten, leiten Sie die Ansichtsklasse aus [CRecordView](../mfc/reference/crecordview-class.md) (für ODBC-Programmierung).|Die Ansicht funktioniert wie eine Formularansicht, aber die Steuerelemente miteinander verbunden sind, auf die Felder einer [CRecordset](../mfc/reference/crecordset-class.md) Objekt, das eine Datenbanktabelle darstellt. MFC verschiebt Daten zwischen den Steuerelementen und das Recordset für Sie.|
|Erstellen Sie einen einfachen Text-Editor.|Wenn Sie die Ansicht, um einen einfachen Text-Editor werden soll, abgeleitet werden, Ihre Ansichtsklasse oder Klassen von [CEditView](../mfc/reference/ceditview-class.md) oder [CRichEditView](../mfc/reference/cricheditview-class.md).|Die Ansicht enthält die Bearbeitung von Funktionen, Unterstützung der Zwischenablage und Datei-e/a. `CRichEditView` Stellt formatierten Text bereit.|
|Fügen Sie Splitterfenster hinzu.|Wenn Sie die Fenster teilen unterstützen möchten, fügen einen [CSplitterWnd](../mfc/reference/csplitterwnd-class.md) -Objekt Ihre SDI-Rahmenfensters oder untergeordneten MDI-Fensters und verknüpfen es mit des Fensters [VIEWEX](../mfc/reference/cframewnd-class.md#oncreateclient) Member-Funktion.|Das Framework Splitter-Steuerelemente neben den Scrollleisten bereitstellt und verwaltet die Ansicht in mehrere Bereiche aufgeteilt. Wenn der Benutzer ein Fenster aufgeteilt ist, wird das Framework erstellt und fügt zusätzliche Objekte an das Dokument.|
|Erstellen, testen und Debuggen der Anwendung.|Verwenden Sie die Funktionen von Visual C++ zu erstellen, testen und Debuggen der Anwendung.|Visual C++ können Sie die Kompilierung, Link und andere Optionen anpassen. Außerdem können Sie die Struktur der Quell- und zu durchsuchen.|
