## <a name="remarks"></a>Bemerkungen

Ein `CMFCToolBarMenuButton` kann als Menü, als Menüelement mit einem Untermenü, einer Schaltfläche, die entweder einen Befehl ausführt oder ein Menü anzeigt, oder als Schaltfläche, die nur ein Menü anzeigt, angezeigt werden. Sie bestimmen das Verhalten und die Darstellung der Menüschaltfläche, indem Sie Parameter wie Bild, Text, Menühandle `CMFCToolbarMenuButton::CMFCToolbarMenuButton`und Befehls-ID angeben, die der Schaltfläche im Konstruktor zugeordnet sind.

Eine von der `CMFCToolbarMenuButton` Klasse abgeleitete benutzerdefinierte Klasse muss das [DECLARE_SERIAL-Makro](run-time-object-model-services.md#declare_serial) verwenden. Das [DECLARE_DYNCREATE-Makro](run-time-object-model-services.md#declare_dyncreate) stritt beim Schließen der Anwendung einen Fehler.

## <a name="cmfctoolbarmenubuttoncmfctoolbarmenubutton"></a><a name="cmfctoolbarmenubutton"></a>CMFCToolBarMenuButton::CMFCToolBarMenuButton

Erstellt ein `CMFCToolBarMenuButton`-Objekt.

```
CMFCToolBarMenuButton();
CMFCToolBarMenuButton(const CMFCToolBarMenuButton& src);

CMFCToolBarMenuButton(
    UINT uiID,
    HMENU hMenu,
    int iImage,
    LPCTSTR lpszText=NULL,
    BOOL bUserButton=FALSE);
```

### <a name="parameters"></a>Parameter

*src*<br/>
[in] Ein `CMFCToolBarMenuButton` vorhandenes Objekt, `CMFCToolBarMenuButton` das in dieses Objekt kopiert werden soll.

*uiID*<br/>
[in] Die ID des Befehls, der ausgeführt werden soll, wenn ein Benutzer auf die Schaltfläche klickt; oder (UINT)-1 für eine Menüschaltfläche, die einen Befehl nicht direkt ausführt.

*Hmenu*<br/>
[in] Ein Handle für ein Menü; oder NULL, wenn die Schaltfläche kein Menü hat.

*iImage*<br/>
[in] Index des Bildes für die Schaltfläche; oder -1, wenn diese Schaltfläche kein Symbol hat oder das Symbol für den von *uiID*angegebenen Befehl verwendet. Der Index ist für `CMFCToolBarImages` jedes Objekt in der Anwendung identisch.

*lpszText*<br/>
[in] Der Text der Symbolleistenmenüschaltfläche.

*bUserButton*<br/>
[in] TRUE, wenn die Schaltfläche ein benutzerdefiniertes Bild anzeigt; FALSE, wenn die Schaltfläche ein vordefiniertes Bild anzeigt, das dem von *uiID*angegebenen Befehl zugeordnet ist.

### <a name="remarks"></a>Bemerkungen

Wenn *uiID* eine gültige Befehls-ID ist, führt die Schaltfläche diesen Befehl aus, wenn der Benutzer darauf klickt. Wenn *hMenu* ein gültiger Menühandle ist, stellt die Schaltfläche ein Dropdown-Menü bereit, wenn sie in einer Symbolleiste oder einem Untermenü angezeigt wird, wenn sie in einem Menü angezeigt wird. Wenn sowohl *uiID* als auch *hMenu* gültig sind, ist die Schaltfläche eine Geteiltschaltfläche mit einem Teil, der den Befehl ausführt, wenn der Benutzer darauf klickt, und einem Teil mit einem Abwärtspfeil, der ein Menü herunterlässt, wenn der Benutzer darauf klickt. Wenn *hMenu* jedoch gültig ist, kann ein Benutzer nicht auf die Schaltfläche klicken, um einen Befehl auszuführen, wenn die Schaltfläche in ein Menü eingefügt wird.

## <a name="cmfctoolbarmenubuttonsetmenuonly"></a><a name="setmenuonly"></a>CMFCToolBarMenuButton::SetMenuOnly

Gibt an, ob die Schaltfläche als Menüschaltfläche oder als geteilte Schaltfläche gezeichnet wird, wenn sie sowohl über eine gültige Befehls-ID als auch über ein Untermenü verfügt.

```

```

### <a name="parameters"></a>Parameter

*bMenuOnly*<br/>
[in] TRUE, um diese Schaltfläche als Menüschaltfläche anzuzeigen, wenn sie sowohl über eine gültige Befehls-ID als auch über ein Untermenü verfügt, FALSE, um diese Schaltfläche als geteilte Schaltfläche anzuzeigen, wenn sie sowohl über eine gültige Befehls-ID als auch über ein Untermenü verfügt.

Wenn der Benutzer in beiden Modi auf den Pfeil klickt, wird das Untermenü geöffnet, und wenn der Benutzer in beiden Modi auf den Nicht-Pfeil-Teil der Schaltfläche klickt, führt das Framework den Befehl aus.
