# <a name="cmultipaneframewnd-class"></a>CMultiPaneFrameWnd-Klasse

Die `CMultiPaneFrameWnd` Klasse erweitert die [CPaneFrameWnd-Klasse](../../mfc/reference/cpaneframewnd-class.md). Es können mehrere Bereiche unterstützt werden. Anstelle eines einzelnen eingebetteten Handles `CMultiPaneFrameWnd` zu einer Steuerelementleiste enthält ein [CPaneContainerManager-Klassenobjekt,](../../mfc/reference/cpanecontainermanager-class.md) das es dem Benutzer ermöglicht, ein sandbares `CMultiPaneFrameWnd` Andocken an einen anderen zu erstellen und dynamisch mehrere unverankerte Fenster mit Registerkarten zu erstellen.

Weitere Informationen finden Sie im Quellcode im **Ordner VC\\atlmfc\\src\\mfc** Ihrer Visual Studio-Installation.

## <a name="syntax"></a>Syntax

```
class CMultiPaneFrameWnd : public CPaneFrameWnd
```

## <a name="remarks"></a>Bemerkungen

Die meisten Methoden in dieser Klasse überschreiben Methoden in der [CPaneFrameWnd-Klassenklasse.](../../mfc/reference/cpaneframewnd-class.md)

Wenn ein Bereich den stilAFX_CBRS_AUTO_ROLLUP verwendet und der Benutzer diesen Bereich an ein Rahmenfenster mit mehreren Fenstern andockt, kann der Benutzer das Fenster unabhängig von den Stileinstellungen der anderen angedockten Bereiche hochfahren.

Das Framework erstellt `CMultiPaneFrameWnd` automatisch ein Objekt, wenn der Benutzer einen Bereich, der den Stil CBRS_FLOAT_MULTI verwendet, verschiebt.

Informationen zum Ableiten einer Klasse `CPaneFrameWnd` aus der Klasse und zum dynamischen Erstellen finden Sie unter [CPaneFrameWnd](../../mfc/reference/cpaneframewnd-class.md).
