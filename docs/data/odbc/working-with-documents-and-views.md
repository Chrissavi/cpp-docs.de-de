# <a name="working-with-documents-and-views"></a>Arbeiten mit Dokumenten und Ansichten

Die MFC-Bibliothek (Microsoft Foundation Classes) basiert auf einer Dokument-/Ansichtsarchitektur für viele ihrer Features. In der Regel speichert ein Dokument Ihre Daten, und eine Ansicht zeigt Sie im Client Bereich eines Rahmen Fensters an und verwaltet die Benutzerinteraktion mit den Daten. Die Ansicht kommuniziert mit dem Dokument, um die Daten abzurufen und zu aktualisieren. Sie können die Datenbankklassen mit dem Framework oder ohne diese verwenden.

Weitere Informationen zum Verwenden von Datenbankklassen im Framework finden Sie unter [MFC: Verwenden von Datenbankklassen mit Dokumenten und Sichten](../../data/mfc-using-database-classes-with-documents-and-views.md).

Der MFC-Anwendungs-Assistent erstellt standardmäßig eine Skeleton-Anwendung ohne Datenbankunterstützung. Allerdings können Sie Optionen auswählen, um minimale Datenbankunterstützung oder eine ausführlichere Formular basierte Unterstützung zu bieten. Weitere Informationen zu den Optionen des Anwendungs-Assistenten finden Sie unter [Datenbankunterstützung, MFC-Anwendungs-Assistent](../../mfc/reference/database-support-mfc-application-wizard.md).

Sie können auch die Datenbankklassen verwenden, ohne die vollständige Dokument-/Ansichtarchitektur zu verwenden. Weitere Informationen finden Sie unter [MFC: Verwenden von Datenbankklassen ohne Dokumente und Sichten](../../data/mfc-using-database-classes-without-documents-and-views.md).
